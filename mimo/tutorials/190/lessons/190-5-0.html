Wouldn't it be great if we were able to turn back time?

If, since committing, we've made a change to cookie.txt that we don't like, we can use checkout to undo it. Let's try!

$ touch cake.txt
$ git add .
$ git commit -m "Add cake"
[master 5508d51] Add cake
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 cake.txt
$ echo Black Forest Cake > cake.txt
$ git checkout cake.txtmaster
$ git status
On branch master
nothing to commit, working tree clean

Fantastic! By checking out the cake.txt file, the changes are removed from the working directory.

Psst: this method doesn't work for changes that are already staged or committed.
