As we've seen, there can be quite some command repetition in Git. Luckily, we can create aliases for commands.

Let's navigate to the $HOME directory, open .gitconfig in a text editor and add an alias section!

$ cd$HOME
$ nano .gitconfig
...
[alias]
  ci = commit
  st = statusversion
...

Woo! With that in place, we can use git ci for commits and git st for the status command.

Psst: .gitconfig is a hidden file that we can only see using ls -a in the Terminal or by telling the Finder to show hidden files.
