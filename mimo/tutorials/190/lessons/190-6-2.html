Let's test our knowledge of branches.

In a repo, create a branch, make a commit (fortune_cookie) there, go back to master, add another commit (cake) and then use the log command to view the commit history of both branches.

$ git branch cool_branch$ git checkout cool_branch$ touch fortune_cookie.txt$ git add fortune_cookie.txt$ git commit -m "Add fortune_cookie.txt"$ git checkout master$ touch cake.txt$ git add cake.txt$ git commit -m "Add cake.txt"$ git log --graph --all --oneline

Phew! Nicely done! It's important you're comfortable with interacting with multiple branches and the commits within them!
