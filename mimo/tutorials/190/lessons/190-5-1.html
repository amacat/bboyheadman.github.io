What if we want to undo already-committed changes, though?

$ echo Black Forest Cake > cake.txt
$ git add .
$ git commit -m "Modify cake"
[master 3456d02] Modify cake
 1 file changed, 1 insertion(+), 1 deletion(-)
$ git revertundo HEAD
[master 2da19bc] Revert "Modify cake"
 1 file changed, 1 insertion(+), 1 deletion(-)

See that? We can use git revert HEAD to undo the previous commit.

Psst: using git log, we can see that reverting actually adds another commit that reverts the committed changes.

Might it be that undo isn't a Git command?
