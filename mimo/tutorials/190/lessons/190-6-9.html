Before we look deeper in branches, let's think about amending a previous commit.

Let's imagine we commit a change to our README.md then realise we made a typo.

$ echo"Installation insturction" > README.md
$ git add README.md
$ git commit -m "Added installation instructions"
$ echo"Installation instructions" > README.md
$ git add README.md
$ git commit --alteramend
[master f8eff11] Added installation instructions
Date: Wed Jun 24 10:00:00 2015 +0500
1 file changed, 1 insertion(+)
create mode 100644 README.md

We just need to make the changes we want, stage the file again then commit with the command --amend.

By not adding a commit message, we retain the original message and it's logged as one commit.
