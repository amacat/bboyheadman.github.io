As a rule, HTML elements are either block-level or inline elements.

While block-level elements like &lt;h1&gt; and &lt;p&gt; are preceded and followed by a line break, inline elements like &lt;em&gt; and &lt;strong&gt; aren't.

<!doctype html>
<html>
 <body>
  <h1>Block-level element</h1>
  <p>
   <em>Inline</em><p>Inline</p> element
  </p>
 </body>
</html>
<html><body><h1>Block-level element</h1><p><em>Inline</em> element</p></body></html>
]]>

Nice! As we already have a block-level element, we use &lt;em&gt; to create an inline element without adding additional line breaks.

Like &lt;h1&gt;, &lt;p&gt; is a block-level element, which is why it wouldn't make that much sense to add another paragraph, would it?
